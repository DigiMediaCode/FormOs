# FormOS Marketing Team Handoff

## Purpose

This document gives a marketing team enough context to understand what FormOS is, who it is for, how to position it, how to sell it, what features matter, and what claims to avoid.

FormOS should not be marketed as another generic form builder. The product direction is:

> FormOS is the form builder that finishes the job.

That means FormOS is for businesses where a form is not the final output. The form starts a workflow that may include customer details, signatures, file uploads, internal staff review, finalized PDFs, organized cloud storage, and website embeds.

---

## 1. Product Overview

### What FormOS Is

FormOS is a SaaS-style workflow form builder for service and rental businesses.

It helps businesses create online forms that can:

- collect customer details
- collect signatures and initials
- collect file uploads
- include Office Use Only fields for staff
- support conditional logic
- generate completed PDF records
- send email notifications
- store uploaded files in Google Drive or Dropbox
- share forms through public links, QR codes, iframe embeds, WordPress, and Shopify
- track basic analytics such as views, submissions, completion rate, and source

### What Problem It Solves

Many small businesses still run important workflows through a mix of:

- paper forms
- PDF agreements
- email attachments
- text messages
- spreadsheets
- cloud folders
- staff notes
- manual PDF creation
- separate e-signature tools

This creates missing files, inconsistent records, slow admin work, and poor customer experience.

FormOS brings these steps into one workflow.

### Core Value Proposition

FormOS helps businesses replace scattered form, signature, upload, internal review, and PDF processes with one organized workflow.

Simple version:

> Collect the details, get the signature, receive the files, complete the internal checks, and generate the final PDF record.

## 7. Permission and Feature Packaging

Feature access is controlled through plan limits and user overrides.

Current feature keys include:

| Feature | Plan/Limit Key | Marketing Meaning |
| --- | --- | --- |
| Number of forms | `maxForms` | Limits how many workflows a user can run. |
| Monthly submissions | `maxMonthlySubmissions` | Controls monthly response volume. |
| Google Drive | `allowGoogleDrive` | Store uploads in Google Drive. |
| Dropbox | `allowDropbox` | Store uploads in Dropbox. |
| PDF generation | `allowPdfGeneration` | Generate completed PDF records. |
| Office fields | `allowOfficeUseFields` | Internal staff-only completion fields. |
| Templates | `allowTemplates` | Start from workflow templates. |
| QR codes | `allowQrCode` | Share forms by QR code. |
| Custom branding | `allowCustomBranding` | Customize owner/workspace branding. |
| Team members | `allowTeamMembers` | Invite staff. |
| Max team members | `maxTeamMembers` | Controls staff seat count. |
| Ad-free forms | `allowAdFreeForms` | Remove public form ads where platform ads are enabled. |
| Embeds | `allowEmbeds` | Embed forms on websites. |
| API access | `allowApiAccess` | External API token access. |
| Conditional logic | `allowConditionalLogic` | Show/hide fields based on previous answers. |
| Conditional rules | `maxConditionalRules` | Limits number of conditional rules. |
| Basic analytics | `allowBasicAnalytics` | Views, submissions, completion rate, source data. |
| Custom submission notifications | `allowCustomSubmissionNotifications` | Send form submission notifications to a custom recipient. |
| Field type control | `allowedFieldTypes` | Controls advanced field access by plan. |

Marketing should avoid promising that every feature is available on every plan.

## 13. Objection Handling

### "We already use Google Forms."

Google Forms is useful for simple forms. FormOS is for workflows that need uploads, signatures, staff-only fields, PDFs, QR codes, embeds, and organized storage.

### "We already use PDFs."

PDFs are familiar, but they usually create manual work. FormOS helps collect, sign, file, review, and finalize the record in one workflow.

### "We need legally binding agreements."

FormOS provides signature and agreement workflow tools, but businesses should review template wording with a qualified advisor for their jurisdiction and use case.

### "Can it work on our website?"

Yes. FormOS supports public links, QR codes, iframe embeds, an embed widget, a WordPress plugin, and a Shopify theme app extension.

### "Where do uploaded files go?"

The form owner can connect Google Drive or Dropbox. Uploaded files are routed to the configured storage provider, and FormOS keeps limited metadata for the submission record.

### "Can staff use it?"

Yes, the workspace system supports team/staff access. Owners can invite staff and keep owner-only areas protected.

### "What happens if we stop paying?"

Existing data is not automatically deleted. Premium actions become limited according to the Free plan or current effective plan.

### "Does FormOS store card details?"

No. Billing is handled through Stripe. FormOS stores billing metadata such as subscription status and customer IDs, not full card details.

## 15. Current Limitations and Roadmap Notes

Be transparent where needed.

- Shopify integration is functional for embeds but not a full App Store-ready app.
- Shopify does not sync products, customers, orders, or payments.
- No advanced analytics dashboard beyond basic views/submissions/completion/source.
- No full conditional workflow engine beyond field visibility logic.
- No full media manager beyond current media library features.
- No per-form role permissions beyond current workspace/staff model.
- No customer-facing support ticket portal.
- No comments/reactions on blog or knowledge base.
